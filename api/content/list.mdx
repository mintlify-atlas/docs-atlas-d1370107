---
title: 'List Content'
api: 'GET /api/v1/content'
description: 'Retrieve all content items for the authenticated user with optional filtering'
---

Fetch all content items belonging to the authenticated user. Results are returned in descending order by creation date (newest first).

## Authentication

This endpoint requires authentication via JWT token.

<CodeGroup>
```bash cURL
curl -X GET "http://localhost:8000/api/v1/content" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```

```bash With Filters
curl -X GET "http://localhost:8000/api/v1/content?type=article&tag=ai" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```

```javascript JavaScript
const response = await fetch('http://localhost:8000/api/v1/content', {
  method: 'GET',
  headers: {
    'Authorization': 'Bearer YOUR_JWT_TOKEN'
  }
});

const data = await response.json();
```

```javascript With Filters
const params = new URLSearchParams({
  type: 'article',
  tag: 'ai'
});

const response = await fetch(
  `http://localhost:8000/api/v1/content?${params}`,
  {
    method: 'GET',
    headers: {
      'Authorization': 'Bearer YOUR_JWT_TOKEN'
    }
  }
);

const data = await response.json();
```

```python Python
import requests

response = requests.get(
    'http://localhost:8000/api/v1/content',
    headers={
        'Authorization': 'Bearer YOUR_JWT_TOKEN'
    }
)

data = response.json()
```

```python With Filters
import requests

response = requests.get(
    'http://localhost:8000/api/v1/content',
    headers={
        'Authorization': 'Bearer YOUR_JWT_TOKEN'
    },
    params={
        'type': 'article',
        'tag': 'ai'
    }
)

data = response.json()
```
</CodeGroup>

## Query Parameters

<ParamField query="type" type="string">
  Filter content by type.
  
  **Options**: `article`, `video`, `tweet`, `document`, `audio`, `image`, `other`
</ParamField>

<ParamField query="tag" type="string">
  Filter content by tag. Tag matching is case-insensitive.
  
  **Note**: Only content items that have this exact tag will be returned.
</ParamField>

## Response

<ResponseField name="message" type="string">
  Success message
</ResponseField>

<ResponseField name="count" type="number">
  Total number of content items returned
</ResponseField>

<ResponseField name="content" type="array">
  Array of content items
  
  <ResponseField name="content[].id" type="number">
    Unique identifier for the content
  </ResponseField>
  
  <ResponseField name="content[].link" type="string">
    The URL of the content
  </ResponseField>
  
  <ResponseField name="content[].type" type="string">
    The content type (article, video, tweet, etc.)
  </ResponseField>
  
  <ResponseField name="content[].title" type="string">
    The content title
  </ResponseField>
  
  <ResponseField name="content[].description" type="string">
    The content description
  </ResponseField>
  
  <ResponseField name="content[].tags" type="array">
    Array of tag strings associated with this content
  </ResponseField>
  
  <ResponseField name="content[].createdAt" type="string">
    ISO 8601 timestamp of when the content was created
  </ResponseField>
</ResponseField>

## Filtering Examples

### Filter by Type

Retrieve only articles:
```bash
GET /api/v1/content?type=article
```

### Filter by Tag

Retrieve content tagged with "ai" (case-insensitive):
```bash
GET /api/v1/content?tag=ai
```

### Combine Filters

Retrieve articles tagged with "ai":
```bash
GET /api/v1/content?type=article&tag=ai
```

## Sorting

Results are always sorted by creation date in descending order (newest first). This sorting is applied after any filters.
