MIND-f-LAYER allows you to share your entire second brain publicly via a unique shareable link. This feature includes endpoints to toggle sharing on/off and to view publicly shared content.

## Toggle Sharing

<api method="POST" url="/api/v1/brain/share" auth="required">
  Enable or disable public sharing of your brain content.
</api>

### Authentication

This endpoint requires authentication. Include a valid JWT token in the Authorization header.

### Request Body

<ParamField body="share" type="boolean" required>
  Set to `true` to enable sharing (creates a new link if none exists), or `false` to disable sharing (removes the share link).
  
  Also accepts string values `"false"` for disabling.
</ParamField>

### Response - Enable Sharing (New Link)

<ResponseField name="message" type="string">
  Status message: `"Share link created successfully"`
</ResponseField>

<ResponseField name="hash" type="string">
  The unique 12-character hash identifier for the share link.
</ResponseField>

<ResponseField name="shareLink" type="string">
  The relative path to access the shared brain: `/api/v1/brain/{hash}`
</ResponseField>

### Response - Sharing Already Active

<ResponseField name="message" type="string">
  Status message: `"Sharing is active"`
</ResponseField>

<ResponseField name="hash" type="string">
  The existing share link hash.
</ResponseField>

<ResponseField name="shareLink" type="string">
  The relative path to the existing shared brain.
</ResponseField>

### Response - Disable Sharing

<ResponseField name="message" type="string">
  Status message: `"Share link disabled successfully"`
</ResponseField>

### Example: Enable Sharing

```bash
curl -X POST http://localhost:8000/api/v1/brain/share \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "share": true
  }'
```

**Response (201 Created):**

```json
{
  "message": "Share link created successfully",
  "hash": "a3f8d9e2b1c4",
  "shareLink": "/api/v1/brain/a3f8d9e2b1c4"
}
```

### Example: Check Existing Share Link

```bash
curl -X POST http://localhost:8000/api/v1/brain/share \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "share": true
  }'
```

**Response (200 OK):**

```json
{
  "message": "Sharing is active",
  "hash": "a3f8d9e2b1c4",
  "shareLink": "/api/v1/brain/a3f8d9e2b1c4"
}
```

### Example: Disable Sharing

```bash
curl -X POST http://localhost:8000/api/v1/brain/share \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "share": false
  }'
```

**Response (200 OK):**

```json
{
  "message": "Share link disabled successfully"
}
```

## View Shared Brain

<api method="GET" url="/api/v1/brain/:shareLink" auth="none">
  Publicly access someone's shared brain content. No authentication required.
</api>

### Path Parameters

<ParamField path="shareLink" type="string" required>
  The unique hash identifier for the shared brain (e.g., `a3f8d9e2b1c4`).
</ParamField>

### Response

<ResponseField name="message" type="string">
  Status message: `"Shared brain content"`
</ResponseField>

<ResponseField name="username" type="string">
  The username of the person who shared this brain.
</ResponseField>

<ResponseField name="count" type="number">
  Total number of content items in the shared brain.
</ResponseField>

<ResponseField name="content" type="array">
  Array of all content items from the shared brain, sorted by creation date (newest first).
  
  <Expandable title="Content Item Object">
    <ResponseField name="id" type="number">
      Unique identifier for the content item.
    </ResponseField>
    
    <ResponseField name="link" type="string">
      URL of the original content.
    </ResponseField>
    
    <ResponseField name="type" type="string">
      Content type: `article`, `video`, `tweet`, `document`, `audio`, `image`, or `other`.
    </ResponseField>
    
    <ResponseField name="title" type="string">
      Title of the content.
    </ResponseField>
    
    <ResponseField name="description" type="string">
      Content description (may be empty string).
    </ResponseField>
    
    <ResponseField name="createdAt" type="string">
      ISO 8601 timestamp when the content was saved.
    </ResponseField>
    
    <ResponseField name="tags" type="array">
      Array of tag strings associated with this content.
    </ResponseField>
  </Expandable>
</ResponseField>

### Example Request

```bash
curl -X GET http://localhost:8000/api/v1/brain/a3f8d9e2b1c4
```

## Implementation Details

### Share Link Generation

- Share links use UUID v4 with hyphens removed and truncated to 12 characters
- Each user can only have one active share link at a time
- Enabling sharing when a link already exists returns the existing link (idempotent)
- Share links are stored in the `links` table with `hash` and `userId` columns

### Security Considerations

<Note>
  **Public Access**: Shared brains are completely public and require no authentication. Anyone with the share link can view all your saved content, including titles, links, descriptions, and tags. Only share your brain if you're comfortable with this level of visibility.
</Note>

- Shared content is read-only - viewers cannot modify or delete content
- The owner's user ID is not exposed in the public response
- Disabling sharing immediately makes the content inaccessible via the previous link
- No analytics or tracking of who views shared brains
