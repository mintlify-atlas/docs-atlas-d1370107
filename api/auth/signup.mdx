## Endpoint

```
POST /api/v1/auth/signup
```

Creates a new user account with the provided credentials. The password is hashed using bcrypt with a salt round of 10 before being stored. Upon successful registration, a JWT token is generated and returned.

## Request Body

<ParamField body="username" type="string" required>
  Username for the account. Must be at least 3 characters long and unique.
</ParamField>

<ParamField body="password" type="string" required>
  Password for the account. Must be at least 6 characters long. Will be hashed using bcrypt with 10 salt rounds before storage.
</ParamField>

## Response

<ResponseField name="message" type="string">
  Success message confirming account creation
</ResponseField>

<ResponseField name="token" type="string">
  JWT authentication token valid for 7 days. Use this token in the Authorization header for authenticated requests.
</ResponseField>

<ResponseField name="userId" type="number">
  The unique identifier assigned to the newly created user
</ResponseField>

## Example Request

```bash
curl -X POST https://api.example.com/api/v1/auth/signup \
  -H "Content-Type: application/json" \
  -d '{
    "username": "johndoe",
    "password": "securepass123"
  }'
```

## Validation Rules

The endpoint uses Zod schema validation with the following rules:

- **Username**: Minimum 3 characters
- **Password**: Minimum 6 characters

## Security Details

### Password Hashing

Passwords are hashed using bcrypt with 10 salt rounds before being stored in the database:

```typescript
const hashedPassword = await bcrypt.hash(password, 10)
```

### JWT Token

The JWT token is signed with a secret key and includes:
- **Payload**: `{ userId: number }`
- **Expiration**: 7 days

### Token Usage

Include the returned token in subsequent API requests:

```bash
curl -X GET https://api.example.com/api/v1/protected-endpoint \
  -H "Authorization: Bearer YOUR_TOKEN_HERE"
```

## Status Codes

- `201` - Account created successfully
- `400` - Validation error (username or password doesn't meet requirements)
- `409` - User already exists with this username
- `500` - Internal server error
