This guide will walk you through setting up MIND-f-LAYER, from getting your API keys to running both the backend and frontend servers.

## Prerequisites

Before you begin, ensure you have:

- **Node.js** (v16 or higher)
- **npm** (comes with Node.js)
- A **Google Gemini API key** (free tier available)

## Get Your Gemini API Key

<Steps>
  <Step title="Visit AI Studio">
    Navigate to [aistudio.google.com](https://aistudio.google.com) and sign in with your Google account.
  </Step>

  <Step title="Create API Key">
    Click on "Get API key" in the left sidebar, then click "Create API key in new project" or select an existing project.
  </Step>

  <Step title="Copy Your Key">
    Once generated, copy your API key. You'll need this in the next step.

    <Note>
      Keep your API key secure! Never commit it to version control or share it publicly.
    </Note>
  </Step>
</Steps>

## Configure Environment Variables

MIND-f-LAYER uses environment variables to manage sensitive configuration. Create a `.env` file in the root of your project:

<Steps>
  <Step title="Create .env file">
    In the `source/` directory, create a new file named `.env`:

    ```bash
    cd source
    touch .env
    ```
  </Step>

  <Step title="Add configuration">
    Open `.env` and add your configuration:

    ```bash .env
    # Gemini AI Configuration
    GEMINI_API_KEY=your_api_key_here

    # JWT Secret (used for authentication)
    JWT_SECRET=your_secure_random_string_here
    ```

    <Tip>
      For `JWT_SECRET`, use a long random string. You can generate one with:
      ```bash
      node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
      ```
    </Tip>
  </Step>

  <Step title="Verify configuration">
    Your environment is now configured! The backend will automatically load these values from `src/configs/config.ts`.
  </Step>
</Steps>

## Install Dependencies

<CodeGroup>

```bash Backend
cd source
npm install
```

```bash Frontend
cd source/frontend
npm install
```

```bash Both (from source/)
npm run build
```

</CodeGroup>

The `npm run build` command installs dependencies for both backend and frontend, then builds the frontend for production.

## Running the Application

### Development Mode

For development, you'll want to run both servers simultaneously in separate terminals.

<Steps>
  <Step title="Start the Backend">
    In your first terminal:

    ```bash
    cd source
    npm start
    ```

    The backend server will start on `http://localhost:8000`. You should see:

    ```
    Server is running on http://localhost:8000
    ```
  </Step>

  <Step title="Start the Frontend">
    In a second terminal:

    ```bash
    cd source/frontend
    npm run dev
    ```

    The frontend development server will start on `http://localhost:5173`. You should see:

    ```
    VITE v7.x.x ready in xxx ms

    ➜  Local:   http://localhost:5173/
    ```
  </Step>

  <Step title="Access the Application">
    Open your browser and navigate to `http://localhost:5173`

    You'll be greeted with the authentication page where you can sign up for a new account.
  </Step>
</Steps>

### Production Mode

For production deployment, the backend serves the built frontend as static files:

```bash
# Build the frontend
cd source/frontend
npm run build

# Start the backend (serves both API and static frontend)
cd ..
npm start
```

The application will be available at `http://localhost:8000`.

## Verify Installation

To verify everything is working correctly:

<Steps>
  <Step title="Create an Account">
    Sign up with a username and password on the authentication page.
  </Step>

  <Step title="Add Test Content">
    Try saving a test link with a title and description.
  </Step>

    If you receive an answer, your Gemini API key is configured correctly!

## Troubleshooting

<AccordionGroup>
  <Accordion title="Backend won't start">
    - Verify you're in the `source/` directory
    - Check that `node_modules/` exists (run `npm install` if not)
    - Ensure port 8000 is not already in use
  </Accordion>

  <Accordion title="Frontend can't connect to backend">
    - Verify the backend is running on port 8000
    - Check the browser console for CORS errors
    - Ensure you're using `http://localhost:5173` (not 127.0.0.1)
  </Accordion>

  <Accordion title="AI queries fail">
    - Verify your `GEMINI_API_KEY` is set correctly in `.env`
    - Check that your API key is valid at [aistudio.google.com](https://aistudio.google.com)
    - Review backend logs for specific error messages
  </Accordion>

  <Accordion title="Database errors">
    The SQLite database is automatically created on first run. If you encounter issues:
    ```bash
    # Remove the database and let it recreate
    rm source/database.sqlite
    npm start
    ```
  </Accordion>
</AccordionGroup>
